# Бізнес-документ ToDo-системи
## Назва системи та короткий опис
ToDo-система — це веб-система для організації завдань. Вона дозволятиме користувачам створювати, редагувати, сортувати та відслідковувати виконання завдань. Система підходить для особистого використання, щоб підвищити ефективність планування та контролю.

Основна мета системи — допомогти користувачам раціонально планувати час, відслідковувати прогрес і зменшити кількість забутих або невиконаних завдань. Система матиме простий та зрозумілий інтерфейс, який забезпечує швидку роботу без зайвих дій.
## Користувачі

## Основні сценарії використання
- Користувач створює нове завдання з описом і дедлайном.  
- Користувач переглядає список своїх завдань і сортує їх за датою і статусом.  
- Користувач редагує або видаляє завдання.  
- Користувач відзначає завдання як виконане. 

## Функціональні вимоги
- Реєстрація та авторизація користувачів.  
- CRUD операції для завдань (створення, читання, редагування, видалення).  
- Встановлення дедлайнів і пріоритетів для завдань.  
- Позначення завдань як виконаних.  
- Фільтрування та сортування завдань.

## Нефункціональні вимоги
- Інтерфейс повинен бути інтуїтивно зрозумілим навіть для нових користувачів.
- Система повинна обробляти стандартні запити користувача за < 2 секунд.
- Користувач має доступ лише до власних завдань
- Дані користувача не передаються третім сторонам.
- Архітектура повинна дозволяти додавання нових модулів без зміни основного коду.

## Обмеження
- Система не підтримуватиме офлайн-режим.
- Не використовуватимуться платіжні сервіси.
